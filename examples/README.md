# 📦 Examples — Dataiku → Fabric Migration Toolkit

This directory contains **real input/output examples** showing exactly what the toolkit processes and produces. Every output file was generated by running the actual agents on the input fixtures.

---

## 📂 Directory Structure

```
examples/
├── README.md                  ← You are here
├── sample_config.yaml         ← Example migration configuration
│
├── input/                     ← Dataiku project assets (what you migrate FROM)
│   ├── project.json           ← Dataiku project metadata
│   ├── recipes/               ← SQL, Python, and visual recipe definitions
│   ├── datasets/              ← Dataset schemas and metadata
│   ├── connections/           ← Dataiku connection configurations
│   ├── flow.json              ← Project flow DAG (dependency graph)
│   └── scenarios/             ← Scheduled automation scenarios
│
└── output/                    ← Fabric assets (what the toolkit produces)
    ├── sql/                   ← Converted SQL scripts (T-SQL)
    ├── notebooks/             ← Generated Fabric Notebooks (.ipynb)
    ├── ddl/                   ← Table DDL statements (warehouse + lakehouse)
    ├── pipelines/             ← Fabric Data Pipeline JSON definitions
    ├── connections/           ← Mapped Fabric connection configs
    └── reports/               ← Validation report
```

### 2. What Each Agent Produces

#### 📝 SQL Migration Agent → `output/sql/`

**Input:** Oracle SQL with `NVL`, `DECODE`, `SYSDATE`, `||`, `LISTAGG`, `ROWNUM`
**Output:** T-SQL with `COALESCE`, `CASE`, `GETDATE()`, `+`, `STRING_AGG`, `ROW_NUMBER()`

```sql
-- BEFORE (Oracle)
SELECT NVL(c.EMAIL, 'unknown@company.com'),
       DECODE(c.STATUS, 'A', 'Active', 'I', 'Inactive', 'Unknown'),
       SYSDATE AS snapshot_date,
       c.FIRST_NAME || ' ' || c.LAST_NAME AS full_name
FROM CRM_CUSTOMERS c

-- AFTER (T-SQL)
SELECT COALESCE(c.EMAIL, 'unknown@company.com'),
       CASE WHEN c.STATUS = 'A' THEN 'Active'
            WHEN c.STATUS = 'I' THEN 'Inactive'
            ELSE 'Unknown' END,
       GETDATE() AS snapshot_date,
       c.FIRST_NAME + ' ' + c.LAST_NAME AS full_name
FROM CRM_CUSTOMERS AS c
```

**Input:** PostgreSQL SQL with `::cast`, `ILIKE`, CTEs, `DISTINCT ON`
**Output:** T-SQL with `CAST(...)`, `LOWER(...) LIKE LOWER(...)`, CTEs, `ROW_NUMBER()`

```sql
-- BEFORE (PostgreSQL)
SELECT DISTINCT ON (product_id) *,
       discount_pct::NUMERIC(5,2),
       NOW() AS computed_at
FROM products
WHERE name ILIKE '%widget%'

-- AFTER (T-SQL)
SELECT product_id, ...,
       CAST(discount_pct AS NUMERIC(5,2)),
       GETDATE() AS computed_at
FROM (SELECT *, ROW_NUMBER() OVER (PARTITION BY product_id ORDER BY ...) AS _rn
      FROM products
      WHERE LOWER(name) LIKE LOWER('%widget%')) _t
WHERE _rn = 1
```

#### 🐍 Python Migration Agent → `output/notebooks/`

**Input:** Python recipe with `dataiku.Dataset`, `dataiku.Folder`, `dataiku.get_custom_variables()`
**Output:** Fabric Notebook with `spark.read.format("delta")`, `spark.conf.get()`

```python
# BEFORE (Dataiku SDK)
import dataiku
df = dataiku.Dataset("CUSTOMER_360_VIEW").get_dataframe()
vars = dataiku.get_custom_variables()

# AFTER (PySpark)
# import dataiku  # Removed — using PySpark native APIs
df = spark.read.format("delta").load("Tables/CUSTOMER_360_VIEW")
vars = spark.conf.get("spark.custom_variables", "{}")
```

#### 📊 Visual Recipe Agent → `output/sql/`

Converts Dataiku visual recipes (JSON-based) into executable SQL:

| Visual Recipe | Generated SQL |
|--------------|---------------|
| **Join** | `SELECT a.*, b.* FROM [CRM_CUSTOMERS] a LEFT JOIN [ERP_ORDERS] b ON a.[CUSTOMER_ID] = b.[CUSTOMER_ID]` |
| **Group By** | `SELECT [REGION], SUM([TOTAL_AMOUNT]) AS [total_revenue], ... GROUP BY [REGION], [PRODUCT_CATEGORY]` |
| **Filter** | `SELECT * FROM ... WHERE [STATUS_LABEL] = 'Active' AND [TOTAL_ORDERS] >= '5'` |
| **Window** | `SELECT *, RANK() OVER (PARTITION BY [REGION] ORDER BY [LIFETIME_REVENUE]) AS [revenue_rank] FROM ...` |
| **Pivot** | `PIVOT (SUM([total_revenue]) FOR [PRODUCT_CATEGORY] IN ([Electronics], [Clothing], ...))` |
| **Prepare** | `UPPER(...)`, `ISNULL(...)`, `REPLACE(...)`, rename/drop/formula transforms |

#### 🗄️ Dataset Agent → `output/ddl/`

Generates `CREATE TABLE` statements targeting either Warehouse (T-SQL) or Lakehouse (Spark SQL):

```sql
-- Warehouse DDL (for Oracle/PostgreSQL datasets)
CREATE TABLE [CRM_CUSTOMERS] (
  [CUSTOMER_ID] BIGINT NULL,
  [EMAIL] NVARCHAR(255) NULL,
  [REGION] NVARCHAR(50) NULL,
  ...
)

-- Lakehouse DDL (for Filesystem datasets, with partitioning)
CREATE TABLE IF NOT EXISTS WEB_SESSIONS (
  [session_id] STRING NULL,
  [SESSION_DATE] DATE NULL,
  ...
) USING DELTA
PARTITIONED BY (SESSION_DATE)
```

#### 🔀 Flow → Pipeline Agent → `output/pipelines/`

Converts the Dataiku flow DAG into a Fabric Data Pipeline JSON with:
- Activity dependency chains (`dependsOn`)
- Script/Notebook activity types
- Retry policies
- Schedule triggers from scenarios

#### 🔗 Connection Mapper → `output/connections/`

Maps each Dataiku connection type to its Fabric equivalent:

| Dataiku | Fabric | Action Required |
|---------|--------|----------------|
| Oracle | OnPremisesDataGateway | Install & configure gateway |
| PostgreSQL | OnPremisesDataGateway | Install & configure gateway |
| S3 | OneLake Shortcut | Create S3 shortcut in OneLake |
| HDFS | OneLake FileCopy | Export → upload to OneLake |
| Filesystem | Lakehouse Files | Upload to Files section |

## 🚀 Try It Yourself

```bash
# 1. Clone the repo
git clone https://github.com/cyphou/DataikuToFabric.git
cd DataikuToFabric

# 2. Install dependencies
pip install -e ".[all]"

# 3. Copy the example config
cp examples/sample_config.yaml config/config.yaml
# Edit with your Dataiku URL + API key and Fabric workspace ID

# 4. Run discovery on your project
dataiku-to-fabric discover --project YOUR_PROJECT

# 5. Run full migration
dataiku-to-fabric migrate --project YOUR_PROJECT

# 6. Check output/ folder for results matching this examples/ structure
```
